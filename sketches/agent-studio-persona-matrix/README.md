## Variant: Persona Matrix

### Design stance
Une matrice responsabilités × étapes qui montre précisément qui intervient, où et pourquoi.

### Key choices
- Layout : personas en lignes, pipeline en colonnes.
- Interaction : drag-and-drop dans chaque cellule, dépliage de Judge, validation simulée.
- Hiérarchie : responsabilité avant chronologie.

### Trade-offs
- Fort : audit des trous, doublons et responsabilités persona par persona.
- Faible : moins fluide pour réordonner les étapes elles-mêmes.

### Best for
Auditer et configurer finement un workflow déjà structuré.
